# 🖥️ IT Assets — Система учёта ИТ-оборудования

## Быстрый старт

### Linux / macOS

```bash
# Клонируй или распакуй папку, затем:
chmod +x start.sh install.sh test.sh

./start.sh      # первый запуск — зависимости подтянутся автоматически
```

Или двумя шагами (явная установка + запуск):
```bash
./install.sh    # однократно — скачивает node_modules
./start.sh      # каждый раз для запуска сервера
```

### Windows

1. Установи **Node.js** с https://nodejs.org (кнопка LTS)
2. Дважды кликни **`INSTALL.bat`** — один раз, скачает зависимости
3. Дважды кликни **`START.bat`** — откроется браузер

## Требования

| Компонент | Минимум  | Рекомендуется |
|-----------|----------|---------------|
| Node.js   | v18 LTS  | v20+ LTS      |
| npm       | v8       | входит в Node |
| ОС        | Windows 10 / Linux / macOS | —  |

### Установка Node.js на Linux

```bash
# Debian / Ubuntu
sudo apt update && sudo apt install nodejs npm

# Fedora / RHEL / Rocky
sudo dnf install nodejs

# Arch Linux
sudo pacman -S nodejs npm

# Через nvm (рекомендуется — без sudo, легко переключать версии)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
# перезапустить терминал, затем:
nvm install --lts
```

## Скрипты

| Скрипт | Windows | Linux/macOS | Назначение |
|--------|---------|-------------|------------|
| Запуск | `START.bat` | `./start.sh` | Запустить сервер (зависимости ставятся сами при первом запуске) |
| Установка | `INSTALL.bat` | `./install.sh` | Явная установка зависимостей |
| Тесты | `TEST.bat` | `./test.sh` | Прогнать тест-сьют |

> **`start.sh` / `START.bat`** — при первом запуске автоматически вызывает `npm install`, если папка `node_modules` отсутствует. Явный вызов `install.sh` не обязателен.

## Вход в систему

Вход осуществляется по логину и паролю — форма появляется при нажатии кнопки **🔐 Войти**.

**Учётная запись по умолчанию:**

| Логин   | Пароль     | Роль          |
|---------|------------|---------------|
| `admin` | `admn0000` | Администратор |

> Смените пароль после первого входа: ⚙️ Настройки → Пользователи.

### Роли

| Роль            | Просмотр | Добавить / изменить | Настройки |
|-----------------|----------|---------------------|-----------|
| Администратор   | ✅        | ✅                  | ✅        |
| Оператор        | ✅        | ✅                  | ❌        |
| Просмотр        | ✅        | ❌                  | ❌        |

Новые пользователи создаются вручную в разделе ⚙️ Настройки → Пользователи. Логин и пароль вводятся при создании.

## Импорт оборудования (CSV)

1. Подготовь CSV с заголовками:  
   `Вкладка, Инв. номер, Филиал, Расположение, Ответственный, Тип, Модель, Серийный №, Статус, Организация, Примечание, MAC, IP, Hostname, Прошивка`
2. Зайди в ⚙️ Настройки → **Импорт из CSV**
3. Выбери файл — система автоматически определит тип (оборудование или история)
4. Нажми **Импортировать**

Дубли по серийному номеру пропускаются автоматически.

## HTTPS и самоподписанный сертификат

Сервер поднимает **два** порта:

| Порт | Протокол | Назначение |
|------|----------|------------|
| `3000` | HTTP  | Только редирект на HTTPS, сам по себе не используется |
| `3443` | HTTPS | Основной адрес приложения |

При первом запуске автоматически генерируется самоподписанный TLS-сертификат (`data/cert.pem` / `data/key.pem`, ~825 дней). Браузер покажет предупреждение о недоверенном сертификате — это ожидаемо:

- **Chrome/Edge:** «Дополнительно» → «Перейти на localhost (небезопасно)»
- **Firefox:** «Дополнительно» → «Принять риск и продолжить»

> Если сертификат сгенерировать не удалось (нет прав на запись в `data/`), сервер автоматически откатывается на обычный HTTP на порту 3000 — об этом будет предупреждение в консоли.

## Доступ из локальной сети

Слушает на `0.0.0.0`. Узнать свой IP:

```bash
# Linux
hostname -I | awk '{print $1}'

# macOS
ipconfig getifaddr en0
```

Сообщите коллегам адрес вида `https://192.168.x.x:3443` (именно `https://` и порт `3443`).

### Открыть порт 3443 в Linux (firewalld / ufw)

```bash
# ufw (Ubuntu/Debian)
sudo ufw allow 3443/tcp

# firewalld (Fedora/RHEL)
sudo firewall-cmd --add-port=3443/tcp --permanent
sudo firewall-cmd --reload
```

## Резервное копирование

Вся база — два файла в папке `data/`:

```bash
cp data/db.json     ~/backup/db_$(date +%Y%m%d).json
cp data/config.json ~/backup/config_$(date +%Y%m%d).json
```

Автобэкап встроен: копии сохраняются в `data/backups/` каждый час (до 30 штук).

## Если что-то не работает

| Проблема | Решение |
|----------|---------|
| `node: command not found` | Установите Node.js (см. раздел «Требования») |
| Белый экран | Откройте DevTools (F12) → Console, скопируйте ошибку |
| Порт занят (`EADDRINUSE`) | HTTP-порт: переменная окружения `PORT` (по умолчанию 3000). HTTPS-порт: переменная `HTTPS_PORT` (по умолчанию 3443). Пример: `PORT=8080 HTTPS_PORT=8443 node server/index.js` |
| Нужен доступ к API с другого домена/фронтенда | Задайте `CORS_ORIGINS` — список разрешённых origin через запятую, например `CORS_ORIGINS=https://intranet.example.com node server/index.js`. По умолчанию кросс-origin запросы из браузера заблокированы (обычному использованию — открыть приложение по своему адресу — это не мешает) |
| Нужно хранить данные не в `data/` рядом с приложением | Переменная `IT_ASSETS_DATA_DIR` указывает путь к папке с `db.json`/`config.json`/`backups/`, например `IT_ASSETS_DATA_DIR=/var/lib/it-assets node server/index.js` |
| Сервер стоит за реверс-прокси (nginx и т.п.) | Задайте `TRUST_PROXY=1`, чтобы rate-limit на логине учитывал реальный IP клиента из `X-Forwarded-For`, а не адрес прокси. Без реверс-прокси эту переменную включать не нужно — иначе клиент сможет подделать заголовок и обойти ограничение попыток входа |
| Медленно открывается | Нормально при первом запуске — БД инициализируется |
| Коллеги не видят | Проверьте firewall (см. выше) |
| `npm install` падает | Попробуйте `npm install --legacy-peer-deps` |
| Не пускает после замены файлов | Перезапустите сервер (`START.bat` / `./start.sh`) |
| Браузер пишет «Соединение не защищено» / `NET::ERR_CERT_AUTHORITY_INVALID` | Ожидаемо — сертификат самоподписанный (см. раздел «HTTPS и самоподписанный сертификат»). Нажмите «Дополнительно» → «Перейти»/«Принять риск» |
| Коллеги не видят / таймаут при подключении | Проверьте, что открыт именно порт **3443** (не 3000) и используется `https://`, а не `http://` |
